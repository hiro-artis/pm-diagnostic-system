## 🏗️ システムアーキテクチャ

### エージェント構成（9個）

```
1. OrchestratorAgent
   └─ テスト流程全体の統制

2. BasicKnowledgeTestAgent
   └─ PM基礎知識テスト（10問）

3. OfficeMigrationTestAgent
   └─ オフィス移転テスト（MC5+Essay3）

4. MindsetTestAgent
   └─ マインドセット検証（6シナリオ）

5. ComprehensiveScorerAgent
   └─ 記述式採点・詳細スコア計算

6. PrimaryTestOrchestratorAgent
   └─ 1次テスト完全フロー制御

7. MindsetInterviewAgent
   └─ マインドセット面接（2次テスト）

8. FinalAssessmentAgent
   └─ 最終評価レポート生成

9. SecondaryTestOrchestratorAgent
   └─ 2次テスト完全フロー制御
```

### 診断フロー

```
[ユーザー開始]
    ↓
[1次テスト] (～60分)
├─ PM基礎知識（10問） → 自動採点 (0-100点)
├─ オフィス移転（5+3問） → 採点 (0-100点)
└─ マインドセット（6シナリオ） → 自動採点 (0-100点)
    ↓
[1次結果判定]
├─ 全テスト合格 かつ マインドセット≥60点
│  → 2次テスト進出
└─ それ以外
   → スキップ
    ↓
[2次テスト] (～20分) ※条件合致時のみ
├─ 弱点マインドセット集中質問（4-6問）
├─ 開放型・深掘り型・検証型質問
└─ 1次テストとの一貫性検証
    ↓
[最終評価] 
├─ 総合スコア計算（重み付け）
├─ 等級判定 (A/B/C/再検査)
├─ マインドセット別分析
└─ 個別推奨事項
    ↓
[レポート提示]
```

## 🔍 システムの特徴

### マインドセット重視設計
- PM適性診断の中心に「6つのマインドセット」を配置
- 単なる知識テストではなく、実務的な判断力・姿勢を評価

### 2段階診断
- **1次テスト（筆記）** - 客観的な知識・判断力評価
- **2次テスト（面接）** - マインドセット深掘り検証

### 個別対応
- 1次テスト結果に基づいて2次テストの質問を最適化
- 弱点マインドセット集中質問

### 透明性
- ルールベースの採点で客観的
- 強み・課題・推奨事項を個別に提示
