### セクション1: PM基礎知識（選択式・10問）
合格基準：70%以上（7問以上）

#### BK-001: プロジェクトの定義
**問題：** 以下の中で、プロジェクトの特性として正しいものはどれか？

A) 継続的で反復的な業務活動  
B) 一時的な努力で独特な成果物を生み出す活動  ✓
C) 組織の日常的な運営活動  
D) 経営層による意思決定プロセス  

**解説：** プロジェクトは期限を持ち、ユニークな成果物を作成する一時的な事業である（PMBOK定義）

---

#### BK-002: ステークホルダー管理
**問題：** プロジェクト成功のために重要でないステークホルダー活動は？

A) ステークホルダーのニーズ理解  
B) 期待値管理  
C) ステークホルダーの関心度・影響度分析  
D) 反対意見のステークホルダーの排除 ✓

**解説：** 反対意見でも、ステークホルダーを巻き込み理解を得ることがPMの重要なスキル

---

#### BK-003: スコープ管理
**問題：** スコープクリープが起こる主な原因は？

A) 厳密なスコープ定義と変更管理の欠如  
B) チームメンバーが十分なスキルを持たない  
C) 正確なスケジュール計画  
D) 効果的なコミュニケーション  

**正答：** A ✓

**解説：** 要件変更を許可制なしに受け入れるとスコープクリープが発生

---

#### BK-004: リスク管理
**問題：** リスク対応戦略の中で、リスク発生確率を低減させる戦略は？

A) 受容（Accept）  
B) 軽減（Mitigate） ✓  
C) 回避（Avoid）  
D) エスカレーション（Escalate）  

**解説：** 軽減は確率または影響を減らす対応。回避は活動そのものを避ける。

---

#### BK-005: コミュニケーション計画
**問題：** 以下の中でコミュニケーション計画で定義する項目ではないのはどれ？

A) コミュニケーション対象者  
B) 情報の内容・粒度  
C) 配信方法・頻度  
D) 配信者の給与水準 ✓

**解説：** 給与は人事・経理の管轄。PMコミュニケーション計画には含まない

---

#### BK-006: 品質管理
**問題：** プロジェクト品質を確保するための活動順序として正しいのは？

A) 検証 → 統合 → テスト  
B) 計画 → 設計 → テスト → 検証 ✓  
C) テスト → 検証 → 統合  
D) 統合 → 検証 → テスト  

**解説：** 計画段階で品質基準を決定し、各フェーズで組み込む必要がある

---

#### BK-007: スケジュール管理
**問題：** 重要パスの特性として正しいのはどれ？

A) 最短で完了できるパス  
B) 最も多くのリソースを使うパス  
C) スラックがなく、遅延するとプロジェクト全体が遅延するパス ✓  
D) 最もコストが高いパス  

**解説：** 重要パスは遅延の余地がないため、重点的に監視する必要がある

---

#### BK-008: コスト管理
**問題：** コスト超過指数（Cost Performance Index：CPI）が0.9の場合、その意味は？

A) コストが10%削減されている  
B) コストが10%超過している ✓  
C) スケジュールが10%遅延している  
D) 品質が10%低下している  

**解説：** CPI = EV/AC。0.9は実際の支出に対して実績値が90%＝超過

---

#### BK-009: 変更管理
**問題：** 変更管理プロセスの目的として最適なものはどれ？

A) すべての変更要望を受け入れる  
B) 変更を可能な限り拒否する  
C) 変更の影響を評価し、承認・拒否を判断する ✓  
D) PМが一人で変更判断を行う  

**解説：** 変更管理委員会（CCB）で適切に評価・判断することが重要

---

#### BK-010: プロジェクト終結
**問題：** プロジェクト終結段階で重要でない活動は？

A) 成果物の引き渡し  
B) 契約の正式終了  
C) プロジェクトメンバーの再配置計画  
D) 過去プロジェクトの実績と比較分析 ✓

**解説：** 過去比較は有用だが、終結段階の必須活動ではない

### MS-S01: 予算削減要請への対応
**マインドセット主軸：未来志向、自責**

**シナリオ：**
プロジェクトが中盤に差し掛かったとき、突然経営層から
「経営環境の変化により、プロジェクト予算を30%削減してほしい」と言われました。

現状：
- 計画通り進捗している
- 当初のスコープ・品質基準は内部で周知済み
- チームメンバーの士気は良好
- 削減されるなら、何らかのスコープ削減は必須

**問題：あなたならどうしますか？**

**A) 経営層の決定に従い、即座に予算削減を受け入れスコープを縮小する**
→ 指示への服従度は高いが、チーム・内部契約者への説明が不足
→ **自責の欠如、未来志向の不足（3点）**

**B) 「計画通り進捗しているので予算削減は難しい」と経営層に報告し、削減を拒否する**
→ 現状報告は適切だが、経営課題への向き合い不足
→ **一人で進まない傾向、未来志向の不足（5点）**

**C) 予算削減が必要な背景・期待を経営層から聴取した上で、
削減下での実現可能なスコープ案・品質維持案を複数作成し提示する** ✓
→ 経営課題の理解、チームへの説明可能性、複数案による柔軟性
→ **未来志向・自責・聴く力を総合的に発揮（15点）**

**D) 予算削減には応じず、代わりに品質を下げることを提案する**
→ スコープ・予算・品質のトレードオフを一方的に決定
→ **聴く力・優しさの欠如（2点）**

**正答：C（15点）**

### MS-S02: チーム内の意見対立への対応
**マインドセット主軸：自責、聴く力**

**シナリオ：**
技術チームと営業チームの意見が対立しています。

技術チーム：「新機能を実装するには最低4週間必要。品質を落とすべきではない」
営業チーム：「顧客は2週間での納期を期待している。実装を急ぐべき」

チーム双方ともあなたのサポートを待っています。経営層の判断も示されていません。

**問題：あなたならどうしますか？**

**A) 技術チームの意見が正しいと判断し、営業チームに納期変更を受け入れるよう指示する**
→ 技術的には正当だが、営業チームの声を聴かない
→ **一人で進まない、置いてきぼりにしない傾向の欠如（4点）**

**B) 営業チームの期待に応えるべく、技術チームに2週間での納期を指示する**
→ 営業の声を聴くが、技術的課題を無視し自責を回避
→ **聴く力は一部あるが、自責・未来志向が不足（3点）**

**C) まず両チームに個別に詳しく話を聴き、
各視点での課題・制約を理解した上で、
技術と営業が一堂に会して解決案を協議する場を設定する** ✓
→ すべての声を聴き、利害関係者を巻き込み、自らが仲介者として関係を構築
→ **自責・聴く力・置いてきぼりにしない・一人で進まないを総合的に発揮（15点）**

**D) 「こういうことがあるプロジェクトだから、双方で妥協してほしい」と曖昧に伝える**
→ 問題をあいまいのままにして解決を避けている
→ **聴く力、自責の欠如（2点）**

**正答：C（15点）**

### MS-S03: 新入メンバーの不安への対応
**マインドセット主軸：優しさ、置いてきぼりにしない**

**シナリオ：**
プロジェクト開始から3ヶ月経った時点で、新しいメンバーがプロジェクトに参加しました。

1週間後、メンバーから「プロジェクトの背景や進捗が十分理解できていない」
「既存メンバーとのコミュニケーションがしづらい」との相談がありました。

既存メンバーは忙しく、新入メンバーへの対応に時間を割く余裕がありません。

**問題：あなたならどうしますか？**

**A) 「プロジェクトは忙しいから、自分で学習するしかない」と伝え、個人の努力に任せる**
→ 置いてきぼりにしない、優しさの完全な欠如
→ **評価スコア：1点**

**B) 既存メンバーに「新入メンバーの対応をお願いします」と指示し、対応を任せる**
→ 新入メンバーの不安に向き合わない。既存メンバーへの配慮も不足
→ **聴く力・優しさの不足（5点）**

**C) 自分の時間を使ってでも新入メンバーとマンツーマンでプロジェクト背景を説明し、
既存メンバーとの交流機会を意図的に作り、
段階的にプロジェクトへの帰属意識を高めるよう支援する** ✓
→ 新入メンバーへの深い優しさ、置いてきぼりにしない姿勢が徹底
→ **優しさ・置いてきぼりにしない・聴く力を総合的に発揮（15点）**

**D) 新入メンバーのためにプロジェクトの進捗を一時停止し、説明会を開催する**
→ 気遣いは見えるが、プロジェクト全体への配慮が欠ける
→ **優しさは示すが、全体最適が欠如（7点）**

**正答：C（15点）**

### MS-S04: 複数部門の相反する要望への対応
**マインドセット主軸：聴く力、一人で進まない**

**シナリオ：**
システム導入プロジェクトで、複数部門から相反する要望が出ています。

営業部：「営業ツール機能を重視したい。納期は3ヶ月」
企画部：「企画プロセスの見える化を重視したい。納期は柔軟」
IT部門：「システム安定性が最優先。追加機能は限定したい」

各部門の責任者は、それぞれの部門の利益を重視しており、譲歩する気配がありません。

**問題：あなたならどうしますか？**

**A) PMの判断で最も重要度の高い部門の要望を採用し、他は後回しにする**
→ 一方的判断で、他の声を聴いていない。一人で進んでいる
→ **聴く力・一人で進まない傾向の完全欠如（2点）**

**B) 各部門から詳しく話を聴き、それぞれの要望の背景・優先順位を理解した上で、
経営層・各部門責任者の合意の下で、優先機能・納期・品質を決定する** ✓
→ すべての声を聴き、最終判断を関係者と共有で進める
→ **聴く力・一人で進まない・自責を総合的に発揮（15点）**

**C) 「全て実装できるように頑張ります」と約束してから、
後で実装不可を伝える**
→ 期待値管理の放棄。置いてきぼりにしない欠如
→ **信頼破壊（0点）**

**D) 各部門の言い分を聴いた後、最大公約数的な要望だけに絞り、
各部門に納得してもらう**
→ 聴く力は示すが、部門の最大関心事を削減している
→ **聴く力は示すが、全体最適性に疑問（8点）**

**正答：B（15点）**

### MS-S05: フロア移転時の意見対立への対応
**マインドセット主軸：置いてきぼりにしない、優しさ**

**シナリオ：**
オフィス移転計画で、一部のメンバー（主に長期勤続者）から
「現在のフロア配置を変えたくない」「新しい部門との協力は難しい」
という強い反発が出ています。

一方、経営層は「部門間連携を強化するために、移転を機にフロア配置を変えるべき」
と決定しています。

反発しているメンバーの意見を黙殺して移転を進めると、
彼らのモチベーション低下や離職の可能性があります。

**問題：あなたならどうしますか？**

**A) 経営層の決定に従い、反発しているメンバーには「会社の決定ですから」と伝える**
→ 反発しているメンバーを置いてきぼりにしている
→ **置いてきぼりにしない・優しさの欠如（3点）**

**B) 反発しているメンバーを新しいフロア配置から除外し、
別のエリアに残す**
→ 問題の先送り。チーム分断。一貫性の欠如
→ **未来志向・一人で進まないが不足（5点）**

**C) 反発しているメンバーと個別に、移転理由・期待・懸念について何度も対話し、
新しい配置に向けた段階的な準備・フォローアップを計画する。
同時に、経営層にも「メンバーの懸念・段階的進行」を報告し、
全体での着地点を整える** ✓
→ すべてのメンバーを巻き込み、懸念に向き合い、納得感を作る
→ **置いてきぼりにしない・優しさ・聴く力・一人で進まないを総合的に発揮（15点）**

**D) 反発しているメンバーの懸念を経営層に報告し、
経営層に判断・説得を任せる**
→ PM自らが仲介・納得形成を放棄している
→ **自責・一人で進まない傾向の欠如（4点）**

**正答：C（15点）**

### MS-S06: 新規領域プロジェクトでの連携
**マインドセット主軸：一人で進まない、未来志向**

**シナリオ：**
あなたは新規事業プロジェクトのPMに抜擢されました。
自分の経験がない領域です。

チーム構成：
- 自分（PM）：営業経験10年、管理経験3年
- 技術リーダー：その道の専門家だが、プロジェクト管理経験は少ない
- ビジネス分析者：初心者で、自分の判断に自信がない
- 外部アドバイザー：専門知識は豊富だが、費用が高い

プロジェクト開始1ヶ月で、当初の計画では成功する見込みが低いことが判明しました。

**問題：あなたならどうしますか？**

**A) 自分のPM経験を活かして、一人で計画を修正し実行する**
→ 専門分野での経験不足。一人で進んでいる。未来への戦略が不十分
→ **一人で進まない・未来志向の欠如（2点）**

**B) 費用を理由に外部アドバイザーを断り、メンバーだけで対応する**
→ 可能な選択肢を消極的に判断。未来志向の不足
→ **未来志向・リソース最適化が不足（6点）**

**C) 技術リーダー・ビジネス分析者・外部アドバイザー（必要に応じて）を交えた
検討会を開催し、各自の知見を引き出しながら、
新しい計画案を共同で作成する。
責任はPMが負いながらも、チーム全体の力を活かす** ✓
→ 自分の不足を認識し、最適なリソースを活用し、チーム全体で未来を描く
→ **一人で進まない・未来志向・聴く力・自責を総合的に発揮（15点）**

**D) 「新規領域なので成功保証はできない」と経営層に報告し、
プロジェクトの延期を提案する**
→ 課題への向き合い不足。自責の欠如
→ **自責・未来志向の欠如（1点）**

**正答：C（15点）**
